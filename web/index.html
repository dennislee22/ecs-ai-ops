Cloudera ECS AI Ops

An air-gapped Kubernetes operations assistant for Cloudera ECS, powered by a local LLM, LangGraph agentic loop, and LanceDB RAG. The LLM can be hosted on GPU or CPU with no code changes required. When no GPU is detected at startup, the application automatically falls back to CPU inference.

⚠ This chatbot is read-only by default. It will never restart, delete, or modify cluster resources. Validated on Qwen3-8B model.

ECS AI Ops Chatbot

Each query runs through an agentic ReAct loop driven by the LLM. The agent selects from a suite of Kubernetes tools covering pod status, node inspection, events, logs, PVC bindings, Longhorn storage, resource metrics, Secrets, deployments, services, ingress, ReplicaSets, and direct database access. It observes the results and decides whether to chain further tool calls or synthesise a final answer. Each query is treated independently with no conversation history.

⚠️ Querying “Which pods are hosted in ecs-m-01.dlee155.cldr.example?” and “List all pods hosted in ecs-m-01.dlee155.cldr.example” yields different interpretations. The latter returns results faster because the chatbot is designed to skip synthesis and directly display the full tool output when it detects “List all,” saving both time and tokens.

Skip Synthesis

The Skip Synthesis toggle bypasses the LLM's final answer generation and streams the raw tool output directly to the chat. Best used for simple single-tool queries where the output is self-explanatory (e.g. logs, pod describe, health scorecard). Responses are typically faster.

⚠️ Enabling this option will show decoded secret values including passwords, tokens, and certificates directly in the chat. Only enable in a secure, private session.

Set Max Output Tokens

Adjust the Max output tokens slider if answers appear incomplete. Increasing it allows the LLM to write longer, more detailed responses. Range: 512 to 8192 tokens (~4 chars/token). Increase if answers are cut off. Higher values allow richer replies but may slow response times.

Settings

Access the Settings panel via the ⚙ Settings button in the header. It is organised into five tabs.

Kubeconfig

Connect the assistant to your Kubernetes cluster. Paste your kubeconfig YAML into the text area and click Apply. The active cluster host is displayed in the header once applied. You can switch clusters at any time with no restart required.

⚠️ Only a single cluster connection is active at a time. Applying a new kubeconfig immediately replaces the previous one.

RAG Documents

Upload and manage documents that feed the LanceDB vector store used by both chatbots. Supported formats: .md, .pdf, .txt, .xlsx, .xls.

  1. Drop files onto the upload zone or click to browse.
  2. Click Ingest. Duplicate content is skipped automatically (hash-based deduplication).
  3. Tick Force re-ingest to overwrite a previously ingested file with updated content. All previously ingested files are listed in the Ingested Files panel at the bottom of this tab.
⚠️ To wipe the entire vector store and start fresh, use the Nuke Vector DB button. This permanently deletes all ingested chunks and uploaded files from disk. Use per-file delete for surgical removal.

Security

Control whether Kubernetes Secret values are shown or redacted in responses. By default, secret values are hidden and only keys are revealed. Toggle visibility in this tab; the preference is saved per browser session.

⚠️ Warning: Enabling this option will show decoded secret values including passwords, tokens, and certificates directly in the chat. Only enable in a secure, private session.

Health Report

Generate an on-demand cluster health snapshot. The report runs a predefined set of diagnostic tool calls covering node status, pod health, PVC bindings, Longhorn volumes, and recent warning events. No LLM synthesis is used. The completed report is automatically compiled and downloaded as a PDF to your browser.

Prompt Guidelines for ECS AI Ops Chatbot

Tips and examples for getting the best responses from the ECS AI Ops Chatbot.

HOW THE CHATBOT DECIDES WHAT TO DO

Every question is independent — there is no conversation history. The chatbot determines your intent and takes one of two paths:

PathTriggered byWhat happens
Live cluster Any operational query — pods, namespaces, secrets, storage, ingress, resources, database… Runs a read-only tool against the live ECS cluster and synthesises the result.
🧠 Model knowledge Must add “don’t run it, just explain” if you want the model to describe what it would do. e.g. what command do you run to get pods in a namespace? Don’t run it, just explain. No tool is called. The model uses its trained knowledge to explain the approach. Without this instruction, operational queries will always execute against the live cluster.
💬 Conversational Greetings, identity questions, general chat Responds directly — no tool is called.

EXAMPLE QUERIES BY CATEGORY

Pods & Health

PODCLUSTER
list all pods in cdp namespace
Lists all pods in the cdp namespace with their phase and ready status.
is the vault pod doing ok?
Checks pod phase, readiness, and recent restart history for any pod named vault.
show all warning events in cdp namespace
Fetches Warning-type Kubernetes events scoped to the cdp namespace, sorted by most recent.
Which pods have restarted more than 5 times in the last 24 hours and why?
Identifies pods that have restarted more than 5 times within the past 24 hours.

Finding Resources

CLUSTER
list any resource named with vault in its name
Searches across pods, deployments, services, statefulsets, ingresses, PVCs, configmaps, and secrets cluster-wide for anything named vault.
Where is grafana?
Searches across all K8s resource types cluster-wide for anything named grafana; including pods, deployments, services, ingresses, and more.

Namespaces

CLUSTER
which namespace has the least pods?
Counts pods per namespace and ranks them.
list all namespaces with total pods
Returns every namespace with its pod count.

Resources & Compute

RESOURCE
list all replicaset in cdp-keda namespace
Lists ReplicaSets in cdp-keda with replica health status.
what's the cpu and ram request for cdp-embedded-db pod in cdp namespace?
Shows CPU and memory requests and limits per container in the pod.
I want to create a pod with 2 vCPU & 8 GB memory, does the cluster have enough available capacity?
Checks allocatable CPU and memory per node against what's already requested, then tells you which node(s) can fit the pod.
calculate CPU requests for all pods in longhorn-system namespace
Aggregates CPU and memory requests/limits for every pod in the namespace in one call.
How much capacity is available to be requested for each node?
Shows the allocatable CPU and memory remaining on each node in %.

Nodes & Taints

CLUSTER
list nodes tainted with "control"
Filters node taints by key/value containing "control" — useful for finding control-plane dedicated nodes.
which node has a GPU available and in use?
Inspects GPU allocatable capacity from the device plugin and shows which pod is currently consuming it.

Networking & Ingress

CLUSTER

Graphs & Metrics

Storage
